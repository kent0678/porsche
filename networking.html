Networking Concepts

Here's how the website works behind the scenes via DNS, IP addresses, and HTTP vs HTTPS

01
DNS — Domain Name System
How your browser finds our website

When you type my website address into the browser, DNS translates that name into an IP address which is the actual numerical location of our server.

I used the nslookup command to prove the domain resolves correctly:

02
IP Addressing — IPv4 & IPv6
The numerical address of the server

IP addresses are the actual locations computers use to find each other on the internet. Our site lives on GitHub's servers at 185.199.108.153 (IPv4). IPv6 is the newer, longer format that supports way more devices.

Type Address Format Total Possible
IPv4 185.199.108.153 32-bit (e.g. 192.168.1.1) 4.3 billion
IPv6 2606:50c0:8000::153 128-bit (much longer) 340 undecillion
03
HTTP vs HTTPS
How data travels between you and our site

HTTP is the basic protocol for sending web pages. HTTPS is the secure version so it encrypts everything so nobody can snoop on the connection. My site enforces HTTPS by the use of GitHub's server by TLS.
