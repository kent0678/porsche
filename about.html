How I Did It

Setup Steps

1
Create a GitHub Repository
I created a public repository named exactly kent0678.github.io. GitHub Pages requires this exact naming to host the website at the root URL.
2
Upload the HTML Files
I added the files index.html, networking.html, and about.html to the main branch of the repository.
3
Enable GitHub Pages
In repository I enabled the pages. The site went live within seconds HTTPS was automatically enabled which meant no extra setup needed.
4
Run Networking Commands
Once the site was live, I ran nslookup, ping, and curl to capture all the networking evidence needed for the report.
nslookup kent0678.github.io & curl -I https://kent0678.github.io/porsche/
What I Learned

Challenges & Solutions

HTTPS Wasn't Fast
GitHub Pages generates an SSL certificate after deployment. There's a window where HTTPS isn't ready yet even if the site loads.
Fix: Waited a few minutes for the certificate to provision, then verified with curl -I to confirm the HTTPS headers appeared.
Understanding the Headers
The curl output showed headers like HSTS and X-Frame-Options that I didn't recognize at first.
Fix: Looked up each header on Google to understand what they do and added explanations to the networking page.
